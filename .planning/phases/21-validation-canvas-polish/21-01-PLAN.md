  <action>
Add 3 new validation rules after the existing Rule 8 (transfer pricing) in `validateGraph()`:

**Rule 9: US S Corp with ineligible shareholder type**
- Only fires for nodes with `entityType === 'us-s-corp'`
- Find all incoming equity edges (edges where target === node.id and relationshipType === 'equity')
- For each shareholder (the source node of each equity edge), look up its config via `getEntityConfig()`
- If the shareholder's category is in `{'company', 'partnership', 'fund', 'holding', 'vc'}`, push a warning: `S Corp has ineligible shareholder type: ${shareholder.data.name} (${shareholderConfig.shortName})`
- If the shareholder's category is `'individual'` AND `shareholderConfig.jurisdiction !== 'US'`, push a warning: `S Corp has non-US individual shareholder: ${shareholder.data.name}`
- Severity: `warning`
- US individuals and US trusts are ELIGIBLE and must NOT trigger the warning
- Import `getEntityConfig` from `@/lib/entity-registry` at the top of the file

**Rule 10: SG VCC without fund manager**
- Only fires for nodes with `entityType === 'sg-vcc'`
- Check if any incoming edge has `relationshipType === 'management'`
- If no management connection exists, push warning: `VCC has no fund manager (management connection)`
- Severity: `warning`

**Rule 11: HK LPF without fund manager**
- Only fires for nodes with `entityType === 'hk-lpf'`
- Check if any incoming edge has `relationshipType === 'management'`
- If no management connection exists, push warning: `Limited Partnership Fund has no fund manager (management connection)`
- Severity: `warning`

Define `S_CORP_INELIGIBLE_CATEGORIES` as a `Set` containing: `'company'`, `'partnership'`, `'fund'`, `'holding'`, `'vc'`. Place it with the other constants near the top of the file (after `TP_RELEVANT_TYPES`).

Update the JSDoc comment at the top of the file to list rules 9-11.

Do NOT modify existing rules 1-8. Do NOT change TRUST_TYPES or PARTNERSHIP_TYPES Sets.
  </action>
