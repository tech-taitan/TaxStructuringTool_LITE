    Export interface `LongPressOptions`:
    - `duration?: number` (default: 500)
    - `moveThreshold?: number` (default: 10 — pixels)
    - `onStart?: () => void` — called immediately on touchstart
    - `onFinish?: () => void` — called when timer fires (long-press complete)
    - `onCancel?: () => void` — called when finger moves beyond threshold or lifts early

    Export function `useLongPress(options: LongPressOptions)` returning object with `onTouchStart`, `onTouchMove`, `onTouchEnd`, `onTouchCancel` handlers (typed as `React.TouchEventHandler<HTMLElement>`).

    Implementation:
    - Use `useRef` for: timer ID (`NodeJS.Timeout | null`), start position (`{x: number, y: number}`), active flag (`boolean`), and a ref to the latest options (to avoid stale closures in the timer callback).
    - Use `useCallback` for each handler for referential stability.
    - `onTouchStart`: record `touch.clientX/Y` as start position, set active=true, call `options.onStart?.()`, start `setTimeout` for `duration` ms that calls `options.onFinish?.()` and sets active=false.
    - `onTouchMove`: compute Euclidean distance from start position. If distance > `moveThreshold`, clear timer, set active=false, call `options.onCancel?.()`.
    - `onTouchEnd` / `onTouchCancel`: if active (timer still pending), clear timer, set active=false, call `options.onCancel?.()`.
    - Cleanup: `useEffect` cleanup clears any pending timer on unmount.

  <action>
    Create `src/lib/spring.ts` — a standalone spring animation utility using requestAnimationFrame.

    Export function `animateSpring`:
    ```typescript
    interface SpringConfig {
      stiffness?: number;   // default: 0.15 — spring pull strength
      damping?: number;     // default: 0.75 — velocity decay per frame
      precision?: number;   // default: 0.5 — stop threshold
    }

    function animateSpring(
      from: number,
      to: number,
      onUpdate: (value: number) => void,
      onComplete?: () => void,
      config?: SpringConfig
    ): () => void   // returns cancel function
    ```

    Implementation:
    - Initialize `position = from`, `velocity = 0`.
    - Each rAF tick: `velocity += (to - position) * stiffness`, `velocity *= damping`, `position += velocity`.
    - Call `onUpdate(position)` each tick.
    - Stop condition: `Math.abs(velocity) < precision && Math.abs(to - position) < precision`. When met, snap to exact `to`, call `onUpdate(to)`, then call `onComplete?.()`.
    - Return a cancel function that calls `cancelAnimationFrame(frameId)`.
    - Use `requestAnimationFrame` for the animation loop (not setInterval).

    Also export a convenience function `animateSpringWithVelocity` for the BottomSheet's velocity-responsive dismiss:
    ```typescript
    function animateSpringWithVelocity(
      from: number,
      to: number,
      initialVelocity: number,
      onUpdate: (value: number) => void,
      onComplete?: () => void,
      config?: SpringConfig
    ): () => void
    ```
    Same as `animateSpring` but seeds `velocity = initialVelocity` instead of 0, making fast swipes feel responsive per the CONTEXT.md decision.

    The file should be ~40-50 lines, pure utility, no React dependencies.
  </action>

  <action>
    Create directory `src/components/mobile/` if it does not exist.

    Create `src/components/mobile/BottomSheet.tsx`:

    **Props interface:**
    ```typescript
    interface BottomSheetProps {
      isOpen: boolean;
      onClose: () => void;
      snapPoints?: ('collapsed' | 'half' | 'full')[];
      initialSnap?: 'half' | 'full';
      children: React.ReactNode;
    }
    ```
    Defaults: `snapPoints = ['collapsed', 'half', 'full']`, `initialSnap = 'half'`.

    **Snap point values (percentage of viewport height from bottom):**
    - `collapsed`: 0% visible (translateY = 100%)
    - `half`: ~45% visible (translateY = 55%)
    - `full`: ~95% visible (translateY = 5%, respecting safe area)

    Use a ref to store the snap point percentages as pixel values computed from `window.innerHeight` (or `dvh` via CSS). Use `transform: translateY()` for all positioning (GPU-composited, no layout thrashing).

    **Rendering structure (using Tailwind):**
    ```
    {isOpen && createPortal(
      <>
        {/* Scrim backdrop */}
        <div className="fixed inset-0 z-[60]"
             style={{ backgroundColor: `rgba(0,0,0,${scrimOpacity})`, pointerEvents: scrimOpacity > 0 ? 'auto' : 'none' }}
             onClick={handleDismiss} />

        {/* Sheet container */}
        <div ref={sheetRef}
             className="fixed inset-x-0 bottom-0 z-[61] bg-white rounded-t-xl shadow-2xl border-t border-gray-200"
             style={{ transform: `translateY(${currentY}px)`, height: '100dvh', touchAction: 'none' }}>

          {/* Drag handle */}
          <div className="flex justify-center pt-3 pb-2"
               onTouchStart={handleTouchStart} onTouchMove={handleTouchMove} onTouchEnd={handleTouchEnd}>
            <div className="w-8 h-1 rounded-full bg-gray-300" />
          </div>

          {/* Content */}
          <div className="overflow-y-auto" style={{ maxHeight: 'calc(100dvh - 48px)' }}>
            {children}
          </div>
        </div>
      </>,
      document.body
    )}
    ```

    **Touch handling (on the entire sheet, not just handle):**
    - Attach touch handlers to the sheet container div (not just the handle pill). The handle area is the primary drag target but the entire top portion should be draggable.
    - `touchstart`: Record startY and current translateY position. Cancel any running spring animation.
    - `touchmove`: Calculate deltaY = currentTouchY - startY. Update `translateY` directly via the ref's style.transform (no state updates during drag for 60fps). Also calculate instantaneous velocity for spring seeding.
    - `touchend`: Compute nearest snap point from current position. If position is below the collapsed point (or velocity is strongly downward with swipe > 50% of collapsed), dismiss the sheet. Otherwise, spring-animate to the nearest snap point using `animateSpringWithVelocity` from `@/lib/spring`, seeding with the drag release velocity.
    - Track velocity: On each touchmove, record `(deltaY - prevDeltaY) / (timestamp - prevTimestamp)` for velocity-responsive spring.

    **Scrim opacity:** Interpolate linearly from 0 (at collapsed position) to 0.4 (at full position). Update during both drag and spring animation.

    **Open/close lifecycle:**
    - When `isOpen` changes to `true`: render the portal, spring-animate from collapsed to `initialSnap` position.
    - When `isOpen` changes to `false` (or `onClose` is called): spring-animate to collapsed, then unmount the portal.
    - Use a local `isRendered` state that stays `true` during the close animation and becomes `false` after the spring completes.

    **Visual design (per CONTEXT.md locked decisions):**
    - Match existing app visual language: `rounded-t-xl` top corners, white background, `border-t border-gray-200`, `shadow-2xl`
    - Handle pill: 32x4px (`w-8 h-1`) `rounded-full bg-gray-300` centered with `pt-3 pb-2`
    - Scrim: semi-transparent dark overlay reaching 30-40% opacity (use 0.4 max)
    - Tap scrim to dismiss

    **Important implementation details:**
    - Use `createPortal` from `react-dom` to render in `document.body` (avoids z-index stacking context issues).
    - Use `100dvh` for the sheet height (handles iOS Safari dynamic toolbar correctly).
    - Set `touch-action: none` on the sheet to prevent browser scroll interference.
    - All position updates during drag MUST use ref.style.transform directly (no React state) for 60fps performance.
    - The scrim div must use `pointer-events: none` when fully transparent to allow clicks through.
