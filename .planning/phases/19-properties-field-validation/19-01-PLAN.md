<task type="auto">
  <name>Task 2: Add Zod validation schemas for all non-AU entity types</name>
  <files>src/lib/validation/entity-schemas.ts</files>
  <action>
Add Zod schemas for all non-AU entity types in `entity-schemas.ts`. Follow the exact pattern from existing AU schemas:
- Each schema extends `baseEntitySchema`
- `entityType` field uses `z.literal()` or `z.enum()` for the entity type IDs
- `registration` uses `.object({...}).optional().default({})`
- `taxStatus` uses `.object({...}).optional().default({})`
- Registration text fields with format validation use `.regex(pattern, message).optional().or(z.literal(''))`
- Registration text fields without format validation use `.string().optional().or(z.literal(''))`
- Boolean fields use `z.boolean().optional()`
- Number fields (rates) use `z.number().min(0).max(1).optional()`
- String select fields use `z.string().optional()`

**Create these reusable field validators first:**
- `ukCompanyNumberField`: `/^[A-Z0-9]{8}$/i` -- "Company Number must be 8 alphanumeric characters"
- `utrField`: `/^\d{10}$/` -- "UTR must be 10 digits"
- `ninoField`: `/^[A-CEGHJ-PR-TW-Z][A-CEGHJ-NPR-TW-Z]\d{6}[ABCD]$/i` -- "NINO must be 2 letters + 6 digits + A/B/C/D"
- `einField`: `/^\d{2}-\d{7}$/` -- "EIN must be in XX-XXXXXXX format"
- `ssnField`: `/^\d{3}-\d{2}-\d{4}$/` -- "SSN must be in XXX-XX-XXXX format"
- `hkCrNumberField`: `/^\d{7,8}$/` -- "CR Number must be 7-8 digits"
- `hkBrnField`: `/^\d{8}$/` -- "BRN must be 8 digits"
- `hkidField`: `/^[A-Z]{1,2}\d{6}[0-9A]$/i` -- "HKID must be letter(s) + 6 digits + check character"
- `uenField`: `/^[A-Z0-9]{9,10}$/i` -- "UEN must be 9-10 alphanumeric characters"
- `nricField`: `/^[STFGM]\d{7}[A-Z]$/i` -- "NRIC/FIN must be letter + 7 digits + letter"
- `rcsNumberField`: `/^[A-Z]\s?\d{1,6}$/i` -- "RCS Number must be letter + up to 6 digits"
- `luNationalIdField`: `/^\d{13}$/` -- "Tax ID must be 13 digits"
