<objective>
Enable mobile connection drawing via a tap-source-then-target flow with connect mode toolbar toggle, floating instruction banner, source entity highlight, and enhanced touch handle visibility.

Purpose: Mobile users currently cannot draw connections between entities. This plan implements the complete tap-to-connect flow using the existing ui-store state machine (mobileTool, pendingConnectionSource) and Canvas connection infrastructure (pendingConnection, MobileConnectionTypePicker). Users enter connect mode via toolbar, tap two entities, pick a type, and the connection is created -- all without any drag precision.

Output: Working mobile connection drawing flow with visual feedback. Covers requirements MCONN-01, MCONN-02, MCONN-03, MCONN-04.
</objective>

**Canvas.tsx -- onNodeClick connect mode intercept:**

1. Subscribe to `mobileTool` from ui-store: `const mobileTool = useUIStore((s) => s.mobileTool);`
2. Subscribe to `pendingConnectionSource` from ui-store: `const pendingConnectionSource = useUIStore((s) => s.pendingConnectionSource);`
3. Modify `onNodeClick` callback to intercept connect mode BEFORE the existing two-step tap logic:
   ```
   if (isMobile && mobileTool === 'connect') {
     if (!pendingConnectionSource) {
       // First tap: set as source
       useUIStore.getState().setPendingConnectionSource(node.id);
     } else if (pendingConnectionSource === node.id) {
       // Tapped same node: deselect source
       useUIStore.getState().setPendingConnectionSource(null);
     } else {
       // Second tap on different node: verify both nodes exist, then initiate connection
       const sourceExists = useGraphStore.getState().nodes.some(n => n.id === pendingConnectionSource);
       if (sourceExists) {
         setPendingConnection({
           source: pendingConnectionSource,
           target: node.id,
           sourceHandle: null,
           targetHandle: null,
         });
       }
       useUIStore.getState().setPendingConnectionSource(null);
     }
     return; // Don't process normal two-step tap logic
   }
   ```
4. Add `mobileTool` and `pendingConnectionSource` to the `onNodeClick` dependency array.
5. Set `connectOnClick` prop on ReactFlow conditionally: `connectOnClick={mobileTool !== 'connect'}`. This disables the built-in handle-tap connection flow when in custom connect mode (prevents conflicts), but leaves it active on desktop and when not in connect mode.
6. Derive `styledNodes` with `useMemo`: when `pendingConnectionSource` is non-null, map over `nodes` and add `className: 'mobile-connect-source'` to the node whose ID matches `pendingConnectionSource`. Pass `styledNodes` to ReactFlow's `nodes` prop instead of raw `nodes`. When `pendingConnectionSource` is null, return nodes unchanged (no unnecessary map).
7. Modify `onPaneClick` to handle connect mode: if `mobileTool === 'connect'` and `pendingConnectionSource` is set, clear `pendingConnectionSource` (deselect source but stay in connect mode). If `mobileTool === 'connect'` but no pending source, do nothing special (stay in connect mode). Only run existing deselect/close logic when NOT in connect mode OR after handling connect mode state. Specifically:
   - At the top of onPaneClick, check: `if (mobileTool === 'connect') { if (pendingConnectionSource) setPendingConnectionSource(null); return; }`
   - This prevents pane clicks from deselecting nodes, closing sheets, etc. while in connect mode.
8. Add `mobileTool` and `pendingConnectionSource` as subscribe selectors (they're already available if subscribed).
