**A. Safe-area CSS utilities (globals.css):**
Add the following utility classes at the END of globals.css (before any final closing bracket):

```css
/* ============================================================
 * Safe Area Inset Utilities (Phase 16)
 * Applied to elements flush with screen edges on notch/
 * home-indicator devices. Requires viewport-fit=cover
 * (already set in layout.tsx).
 * ============================================================ */

.safe-area-bottom {
  padding-bottom: env(safe-area-inset-bottom, 0px);
}

.safe-area-top {
  padding-top: env(safe-area-inset-top, 0px);
}
```

Only `safe-area-bottom` and `safe-area-top` are needed — left/right are not applicable because all mobile UI elements are centered or full-width.

**B. Mobile bottom toolbar (MobileEditorLayout.tsx):**
The toolbar at line 92 uses `className="absolute bottom-4 ..."`. On notch devices, `bottom-4` (16px) is NOT enough to clear the home indicator (~34px). Change the `bottom-4` positioning to use a CSS `style` prop that accounts for the safe-area-inset-bottom:
- Add `style={{ bottom: 'max(16px, calc(env(safe-area-inset-bottom, 0px) + 8px))' }}` to the toolbar div.
- Remove `bottom-4` from the className (it would conflict with the inline style).
- Keep the rest of the className unchanged.

**C. MobilePalette dead class fix (MobilePalette.tsx):**
On line 99, replace `className="pb-safe"` with `className="safe-area-bottom"`. The `pb-safe` class is UNDEFINED (not a Tailwind CSS v4 built-in, no safe-area plugin installed). The new `safe-area-bottom` class defined in globals.css provides `padding-bottom: env(safe-area-inset-bottom, 0px)`.

**D. BottomSheet content area (BottomSheet.tsx):**
