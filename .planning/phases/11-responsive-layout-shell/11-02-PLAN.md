  <action>
**In Canvas.tsx:**

1. Import `useDeviceCapabilities` from `@/hooks/useDeviceCapabilities`. Extract `isTouchDevice`, `isMobile`.

2. Add touch-specific React Flow props to the `<ReactFlow>` element:
   - `zoomOnPinch={true}` -- always true, React Flow handles pinch-to-zoom natively
   - `preventScrolling={true}` -- prevent page scroll when interacting with canvas
   - `panOnDrag` -- on touch devices: always `true` (context-sensitive: touching empty canvas pans, touching entity moves it -- React Flow handles this natively when panOnDrag is true and nodes are draggable). On desktop: keep existing `interactionMode === 'drag'` behavior unchanged.
     Expression: `panOnDrag={isTouchDevice ? true : interactionMode === 'drag'}`
   - `selectionOnDrag` -- on touch: `false` (no marquee selection on touch). On desktop: keep existing behavior.
     Expression: `selectionOnDrag={isTouchDevice ? false : interactionMode === 'select'}`
   - `selectionKeyCode` -- on touch: `null` (disable shift-selection concept). On desktop: keep existing.
   - `nodesDraggable={true}` -- ensure nodes remain draggable (default, but explicit for clarity)

3. Disable double-click-to-create-entity on touch devices:
   - Change `onDoubleClick={onDoubleClickPane}` to `onDoubleClick={isTouchDevice ? undefined : onDoubleClickPane}`
   - This prevents accidental entity creation from double-tap, which is confusing on touch. Entity creation on mobile will be via FAB in Phase 12.

4. Conditionally hide desktop-only UI elements inside the ReactFlow component:
   - `<Controls />` -- wrap in `{!isMobile && <Controls />}`. Controls (zoom in/out/fit) are replaced by pinch-to-zoom on mobile and will be in the toolbar for tablet.
   - `<MiniMap>` -- wrap in `{!isMobile && <MiniMap ... />}`. MiniMap takes precious screen space on phone. Tablet gets it in Phase 15.
   - `<HelperLines>` -- wrap in `{!isTouchDevice && <HelperLines ... />}`. Alignment snap guides are a precision feature not useful on touch.

5. Hide StatusBar on mobile in EditorLayout:
   - Actually, StatusBar is rendered in EditorLayout.tsx, not Canvas.tsx. The mobile path already bypasses it because MobileEditorLayout (from Plan 01) does not render StatusBar. No change needed here -- just verify.

6. Disable context menus on touch (they are right-click based):
   - Change `onNodeContextMenu` to `{isTouchDevice ? undefined : onNodeContextMenu}` in the ReactFlow props
   - Change `onEdgeContextMenu` to `{isTouchDevice ? undefined : onEdgeContextMenu}` in the ReactFlow props
   - Long-press context menu is handled in Phase 13.

7. Disable keyboard shortcuts on mobile (no keyboard):
   - The `useKeyboardShortcuts` hook can stay active -- keyboard events simply won't fire on touch-only devices. No change needed.

**In layout.tsx:**

Add a viewport meta tag to the `<head>` section. In Next.js App Router, use the `metadata.viewport` export:

```typescript
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  maximumScale: 1,
  userScalable: false,
  viewportFit: 'cover',
};
```

Import the `Viewport` type from `next`. This prevents browser-level pinch-to-zoom (which would zoom the whole page, not the canvas), and `viewport-fit: cover` enables safe area insets for notch devices. The `maximumScale: 1` and `userScalable: false` ensure pinch gestures go to React Flow, not the browser.

**In globals.css:**

Add a `@media (pointer: coarse)` section for touch handle styling:

```css
/* ============================================================
 * Touch Device Handle Overrides (Phase 11)
 * On touch devices: handles always subtly visible, enlarged to
 * 44px hit area minimum per WCAG/Apple HIG touch target guidance.
 * ============================================================ */

@media (pointer: coarse) {
  /* Make handles subtly visible on touch (no hover to discover them) */
  .react-flow__handle {
    opacity: 0.3;
  }

  /* Enlarge invisible hit area to 44px minimum */
  .react-flow__handle::after {
    top: -15px;
    left: -15px;
    right: -15px;
    bottom: -15px;
  }

  /* When node is selected, handles fully visible */
  .react-flow__node.selected .react-flow__handle {
    opacity: 0.7;
  }

  /* Entity node touch feedback */
  .entity-node:active {
    transform: scale(0.98);
  }
}
```

Explanation of the 44px hit area math:
- Handle is 14px wide/tall (from existing CSS)
- `::after` extends -15px each side: 15 + 14 + 15 = 44px effective touch target
- This meets the 44x44px minimum touch target from Apple HIG and WCAG 2.5.5

Also add handle visibility on node hover/selection (for when handles go from opacity 0.3 to visible on interaction):

```css
@media (pointer: coarse) {
  /* During active connection drawing, show all handles clearly */
  .react-flow__node.connecting .react-flow__handle {
    opacity: 0.8;
  }
}
```
