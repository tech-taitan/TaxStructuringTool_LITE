**MobileContextMenu.tsx -- new component:**
- Create `src/components/mobile/MobileContextMenu.tsx`.
- Props: `x: number`, `y: number`, `nodeId: string`, `onClose: () => void`.
- The component reads actions from the parent or provides its own action handlers:
  - **Delete**: Import `useUIStore` and `useGraphStore`. Call `setDeleteConfirm({ nodeIds: [nodeId], connectionCount })` where connectionCount is derived from edges containing nodeId. Then call `onClose()`.
  - **Copy**: Import `useClipboard`. First, select the target node in graph store via `useGraphStore.getState().onNodesChange([{ id: nodeId, type: 'select', selected: true }])`. Then call `copy()`. Then call `onClose()`.
  - **Connect**: Import `useUIStore`. Call `setPendingConnectionSource(nodeId)` and `setMobileTool('connect')`. Then call `onClose()`. (Wires Phase 14 connect flow.)
  - **Properties**: Call `setMobilePropertiesOpen(true)`. Then call `onClose()`. (Node is already selected by long-press.)
- Render a `fixed` positioned `div` with `z-50`, styled as a floating card: `bg-white rounded-xl shadow-lg border border-gray-200 py-1 min-w-[180px]`.
- Position using `style={{ top: clampedY, left: clampedX }}`.
- **Position clamping logic:** Use a `useEffect` + `useRef` pattern:
  1. Render the menu offscreen initially (or at the raw x,y).
  2. In a `useLayoutEffect`, measure the menu with `getBoundingClientRect()`.
  3. Clamp: `x` stays within `[16, window.innerWidth - menuWidth - 16]`. `y` stays within `[16, window.innerHeight - menuHeight - 16]`. If press point is in the bottom half of the screen, position menu ABOVE the press point (`y - menuHeight - 16`); otherwise below (`y + 16`).
  4. Update a `pos` state with clamped coordinates.
- Four action buttons, each with `min-h-[44px]` for touch targets:
  - `<Trash2 /> Delete` (text-red-600)
  - `<Copy /> Copy` (text-gray-700)
  - `<Link /> Connect` (text-gray-700)
  - `<Settings /> Properties` (text-gray-700)
- Each button: `onClick` calls the action and then `onClose()`.
- **Dismiss on tap outside:** Add a `useEffect` that listens to `document touchstart` (NOT mousedown -- this is mobile only). If the touch target is not inside the menu ref, call `onClose()`. Use `{ passive: true }` for performance.
- Use `lucide-react` icons: `Trash2`, `Copy`, `Link`, `Settings`.
- Add subtle entrance animation via CSS class: `animate-in fade-in-0 zoom-in-95` (or a simple opacity/scale CSS transition). If Tailwind animate utilities are not available, add a simple `@keyframes context-menu-in` to globals.css.

**EntityNode.tsx -- integrate long-press using combined wrapper approach:**
- Import `useLongPress` from `@/hooks/useLongPress`.
- Import `useDeviceCapabilities` (already imported).
- Import `useUIStore` for `setMobileContextMenu`, `setSelectedNode`.
- Add a `touchCoordsRef = useRef({ x: 0, y: 0 })` to capture touch position.
- Create long-press handlers using `useLongPress` with these callbacks:
  - `onStart`: no-op (coordinate capture happens separately).
  - `onFinish`: (1) select the node: `useUIStore.getState().setSelectedNode(nodeId)` (also `useGraphStore.getState().onNodesChange([{ id: nodeId, type: 'select', selected: true }])`), (2) set context menu: `useUIStore.getState().setMobileContextMenu({ x: touchCoordsRef.current.x, y: touchCoordsRef.current.y, nodeId })`, (3) haptic feedback: `navigator.vibrate?.(10)`.
  - `onCancel`: no-op (drag takes over).
- **Use the combined wrapper approach for touch handlers (do NOT "spread handlers onto the outermost div" directly):** Define a combined `onTouchStart` handler that:
  1. Captures `e.touches[0].clientX/clientY` into `touchCoordsRef`.
  2. Delegates to `longPressHandlers.onTouchStart(e)`.
- Similarly, pass through `longPressHandlers.onTouchMove`, `longPressHandlers.onTouchEnd`, and `longPressHandlers.onTouchCancel` directly (no wrapping needed for those).
- Attach these combined handlers to the outermost div of EntityNode: `onTouchStart={combinedTouchStart}`, `onTouchMove={longPressHandlers.onTouchMove}`, etc.
- **Only attach long-press handlers when `isTouchDevice` is true.** Use conditional spreading: `{...(isTouchDevice ? touchHandlerProps : {})}` where `touchHandlerProps` is an object with the four handlers above.
- Ensure the long-press gesture does NOT conflict with React Flow's built-in node drag. The `useLongPress` hook cancels if finger moves >10px (its `moveThreshold`), so a drag gesture will cancel the long-press. The long-press fires after 500ms of stillness, at which point React Flow hasn't started dragging (it uses a drag threshold too).
