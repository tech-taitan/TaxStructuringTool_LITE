**BottomSheet.tsx -- add imperative ref API:**
- Add `React.forwardRef` to the BottomSheet component (or use `useImperativeHandle` with a new `ref` prop).
- Export a `BottomSheetRef` interface with `snapTo(snap: 'collapsed' | 'half' | 'full'): void` method.
- Implement `snapTo` by calling the internal `springTo(SNAP_PERCENTS[snap])` function.
- Export the `SNAP_PERCENTS` constant so the keyboard hook can reference snap point values directly.
- Keep ALL existing behavior unchanged -- the ref API is additive only.

**useKeyboardAwareSnap.ts -- new hook:**
- Create `src/hooks/useKeyboardAwareSnap.ts`.
- Hook signature: `useKeyboardAwareSnap(isOpen: boolean, sheetRef: React.RefObject<BottomSheetRef | null>)`.
- On mount (when `isOpen` becomes true), capture `window.visualViewport.height` as the baseline.
- Listen to `window.visualViewport` `resize` event.
- When `initialHeight - visualViewport.height > 100` (keyboard threshold), call `sheetRef.current?.snapTo('full')`.
- When height difference drops back below threshold, call `sheetRef.current?.snapTo('half')`.
- Track keyboard state in a ref (`isKeyboardOpenRef`) to avoid redundant snap calls.
- Guard with `if (!window.visualViewport) return` for SSR/non-Safari safety.
- Cleanup: remove event listener on unmount or when `isOpen` becomes false. Reset `isKeyboardOpenRef` to false when `isOpen` changes to false.

**ui-store.ts -- add mobileContextMenu state:**
- Add `mobileContextMenu: { x: number; y: number; nodeId: string } | null` field, initially `null`.
- Add `setMobileContextMenu: (menu: { x: number; y: number; nodeId: string } | null) => void` setter.
- Add `isMobileAnalysisOpen: boolean` field, initially `false`.
- Add `setMobileAnalysisOpen: (open: boolean) => void` setter.
- Add these to the UIState interface and the store initializer.

**Canvas.tsx -- add two-step tap for mobile properties:**
- Import `useDeviceCapabilities` (already imported) and add mobile properties logic.
- Add an `onNodeClick` handler: when `isMobile`, read current `selectedNodeId` from `useUIStore.getState()`. If `selectedNodeId === node.id` (second tap on same node), call `useUIStore.getState().setMobilePropertiesOpen(true)`. If it's a different node and properties sheet is already open, sheet stays open (content re-renders via key). React Flow handles the selection change automatically via `useOnSelectionChange`.
- Add an `onEdgeClick` handler: when `isMobile`, read current `selectedEdgeId` from `useUIStore.getState()`. If `selectedEdgeId === edge.id` (second tap on same edge), call `useUIStore.getState().setMobilePropertiesOpen(true)`.
- Pass `onNodeClick` and `onEdgeClick` to the `<ReactFlow>` component.
- Modify `onPaneClick`: when `isMobile`, also call `setMobilePropertiesOpen(false)` and `setMobileContextMenu(null)` to close sheets/menus on canvas tap.
- For the connection type picker, add conditional rendering: when `pendingConnection` exists and `isMobile`, render `MobileConnectionTypePicker` instead of `ConnectionTypePickerModal`. Import `MobileConnectionTypePicker` (will be created in Task 2 -- use dynamic import or just import directly; the file will exist by task execution order).

**IMPORTANT -- Do NOT add onNodeClick/onEdgeClick if they conflict with existing handlers. The existing `onNodeContextMenu` is already conditional on `!isTouchDevice`. The new `onNodeClick`/`onEdgeClick` are separate React Flow callback props that do not conflict.**
